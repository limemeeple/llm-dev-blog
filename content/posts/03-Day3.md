## 클라이언트의 요구사항 변경!

클라이언트(아내)에게 개발 진행 상황 및 결과를 전달하던 과정에서 갑자기 새로운 요구사항을 전달받게 되었다.

**파일 보안 때문에 로컬에서 작업할 수 있어야 해!**

뭐야! 그런 말 없었잖아!!

지금까지 Claude API를 사용해서 개발하던 방향성을 틀어야 했다.

로컬로 구현이 가능한지 알아보자 'Ollama'라는걸 찾게 되었다.

## Ollama란 무엇인가?

복잡한 설치 과정없이 로컬 환경에서 LLM을 쉽게 실행할 수 있게 도와주는 오픈소스라고 한다.
특히 설치가 쉽기 때문에 초보자들이 사용하기 좋다고하여 선택하였다.

[Ollama의 장점]
- 프라이버시 보호
- 오프라인 사용
- 비용 절감
- 간편한 사용

[Ollama의 단점]
- 높은 컴퓨터 사양 요구(하드웨어 성능)
- AI 모델의 성능 한계
- 컴퓨터 자원 독점 및 발열
- 텍스트 위주의 기본 환경


## Ollama 교체 작업

**ollama.com** 에서 윈도우용 설치 파일을 설치하면 된다.(백그라운드에서 도는 프로그램이라 특별히 실행할 필요는 없음.)

설치 이후에 'cmd'에서 모델을 받는다.
```
ollama pull qwen2.5:7b
```
[추천 받은 이유]
- qwen2.5 계열은 한국어를 잘함.
- 7b 사양 부담이 적당 함.
- 사양이 낮으면 'qwen2.5:3b' 시작해도 되고 여유가 있으면 'qwen2.5:14b' 품질을 올린다.

현재로 비교해 봤을때 Qwen3:8B도 추천받았다 이 2개의 모델을 비교해봤다.

[Qwen3:8B vs Qwen2.5:7B 비교표] 

|항목|Qwen2.5 (7B)|Qwen3 (8B)|
|----|----|----|
|추론 능력|준수함 (일상 대화용)|매우 뛰어남 (복잡한 논리)|
|한국어자연스러움|양호|매우 자연스러움|
|코딩 성능|우수함|최상급 (디버깅 특화)|
|속도 (Response)|빠름|유사함 (살짝 더 무거울 수 있음)|
|메모리(RAM) 요구량|8GB 권장|8GB ~ 12GB 권장|
|특이 사항|안정적인 범용 모델|Thinking(사고) 기능 지원|

[요약 및 추천]
- Qwen2.5:7B: 지금 내 컴퓨터에서 가장 빠르고 가볍게 일상적인 대화나 간단한 텍스트 작업을 하고 싶을 때.
- Qwen3:8B: 코딩 보조, 복잡한 문제 해결, 더 똑똑한 한국어 작문 등 '성능'이 조금 더 중요할 때.

테스트를 진행해본다.
```
ollama run qwen2.5:7b "안녕, 한국어로 대답해줘"
```
한국어로 답변 받으면 준비 끝이다.

## 조견표 만들어 보자!

슬라이드/페이지 단위로 행을 만들고, 각 행의 요약을 Ollama로 뽑는 조견표를 만들기로 했다.

ex>
|파일명|위치|원문(일부)|한 줄 요약|
|----|----|----|----|
|사업계획.pptx|슬라이드 1|2024년 지원사업 추진계획...|2024년 지원사업 전체 개요|
|사업계획.pptx|슬라이드 2|재택근무는 주 3일까지...|재택근무 운영 기준 안내|
|예산.pdf|페이지 1|총 예산 15억원 편성...|연간 예산 편성 내역|

[조견표 만드는 코드]
```
"""
make_index.py — 폴더 안 모든 자료를 슬라이드/페이지 단위로 훑어
'조견표'(파일명·위치·원문·요약)를 엑셀로 만든다.

요약은 Ollama(로컬)로 생성한다. 인터넷 불필요.
실행: python make_index.py
"""

import os
os.environ["HF_HUB_DISABLE_SYMLINKS"] = "1"

import json
from pathlib import Path
from openpyxl import Workbook
from openpyxl.styles import Font, PatternFill, Alignment

import ollama
from rag import read_pptx, read_pdf, read_xlsx


# ──────────────────────────────────────────────────────────────
# 설정
# ──────────────────────────────────────────────────────────────
DATA_FOLDER = r"C:\llm-rag-qa\data"        # 조견표로 만들 자료 폴더
OUTPUT_XLSX = r"C:\llm-rag-qa\조견표.xlsx"
OLLAMA_MODEL = "qwen2.5:7b"
LIMIT = 0          # 0이면 전체. 테스트할 땐 10 정도로 두면 앞 10개만 처리
MIN_CHARS = 15     # 이보다 짧은 블록은 요약 안 하고 원문만


def read_any(path):
    """확장자에 맞게 파일을 읽어 (블록리스트, 위치라벨)을 돌려준다."""
    suffix = path.suffix.lower()
    if suffix == ".pptx":
        return read_pptx(path), "슬라이드"
    elif suffix == ".pdf":
        return read_pdf(path), "페이지"
    elif suffix == ".xlsx":
        return read_xlsx(path), "시트"
    return None, None


def summarize(text):
    """Ollama로 한 줄 요약을 만든다."""
    prompt = f"""다음 내용을 한국어로 짧게 한 줄(20자 내외) 요약해줘.
제목처럼 명사구로. 다른 말 붙이지 말고 요약만.

내용: {text[:1000]}"""
    try:
        response = ollama.chat(
            model=OLLAMA_MODEL,
            messages=[{"role": "user", "content": prompt}],
        )
        return response["message"]["content"].strip().replace("\n", " ")
    except Exception as e:
        return f"(요약 실패: {e})"


def save_to_excel(rows, path):
    wb = Workbook()
    ws = wb.active
    ws.title = "조견표"

    headers = ["파일명", "위치", "원문(일부)", "한 줄 요약"]
    ws.append(headers)
    for cell in ws[1]:
        cell.font = Font(bold=True, color="FFFFFF", name="맑은 고딕")
        cell.fill = PatternFill("solid", start_color="4472C4")
        cell.alignment = Alignment(horizontal="center", vertical="center")

    for r in rows:
        ws.append(r)

    widths = [24, 12, 60, 30]
    for col, w in enumerate(widths, start=1):
        ws.column_dimensions[ws.cell(row=1, column=col).column_letter].width = w
    for row in ws.iter_rows(min_row=2):
        for cell in row:
            cell.alignment = Alignment(vertical="top", wrap_text=True)

    wb.save(path)


if __name__ == "__main__":
    folder = Path(DATA_FOLDER)
    print(f"📂 '{folder}' 폴더를 훑는 중...\n")

    # 먼저 모든 블록을 모은다 (요약 전)
    entries = []   # (파일명, 위치, 원문)
    for path in folder.rglob("*"):
        blocks, label = read_any(path)
        if not blocks:
            continue
        for idx, block in enumerate(blocks, start=1):
            block = block.strip()
            if block:
                entries.append((path.name, f"{label} {idx}", block))
        print(f"   ✅ {path.name} → {len(blocks)}개")

    # 테스트용 제한
    if LIMIT > 0:
        entries = entries[:LIMIT]
        print(f"\n   (테스트 모드: 앞 {LIMIT}개만 처리)")

    print(f"\n📄 총 {len(entries)}개 항목을 요약합니다. (시간이 걸려요)\n")

    # 요약 달기
    rows = []
    for n, (fname, loc, text) in enumerate(entries, start=1):
        if len(text) >= MIN_CHARS:
            summary = summarize(text)
        else:
            summary = text[:20]   # 너무 짧으면 요약 대신 원문
        print(f"   ({n}/{len(entries)}) {fname} {loc} → {summary}")
        rows.append([fname, loc, text[:400], summary])

    save_to_excel(rows, OUTPUT_XLSX)
    print(f"\n💾 조견표를 저장했습니다: {OUTPUT_XLSX}")
```
- read_any : 확장자를 보고 알맞은 읽기 함수로 보내주는 교통정리 함수, 'load_sentences'와 비슷한 역할 이지만 문장으로 쪼개지 않기 때문에 사용.

- summarize : Ollama로 한줄 요약을 만드는 부분.

- LIMIT : 후에 0으로 바꿔서 전체 요약하도록 해야한다. 지금은 품질 확인을 위해 앞에 10개만 요약해보도록 적용.
 
## 문제점 확인.(불필요한 임배딩 작업 실행.)

현재 조견표 생성, 중복체크 기능을 사용할 때 rag.py에 있는 특정 폴더 임배딩 기능이 같이 돌아가는걸 확인했다.

확인해보니 rag 파일을 임포트 한후 read_pptx, read_pdf, read_xlsx 함수를 가져올때 rag.py를 통째로 실행한다는걸 알았다.

그 중 임배딩하는 코드가 함수에 정의된게 아니라 바깥에 나와있기때문에 불필요한 작업이 일어나고 있었던 것이다. 그래서 해당 기능을 함수로 묶어서 더 이상 불필요한 작업이 일어나지 않도록 수정했다.

## 두번째 문제점 확인.(조견표 내용이 만들어 지지 않는 문제.)

조견표 생성 기능을 만들기 테스트를 위해 교육용으로 나눠준 pdf 자료를 폴더에 넣고 돌려보았는데 내용이 비어있는 엑셀 자료를 만들었다.

원인 파악을 위해 차례대로 확인작업에 들어갔다.

[원인 파악]
아래와 같은 한줄 명령어를 통해 PDF 내용을 잘 읽어오는지 확인해 봤다.
```
python -c "from rag import read_pdf; r = read_pdf(r'경로\파일.pdf'); print('블록 수:', len(r))"
```
결과 블록 수가 0으로 나오는걸 확인했다. 하지만 에러는 발생하지 않는다. 이 부분에 대해 파트너에게 확인 요청을 해봤다.

[원인]
원인은 **PDF가 스캔한 PDF** 였기 때문이었다. 현재 pdf의 내용을 추출하는 'pypdf'는 텍스트를 추출하는 도구라서 그림 안의 글자는 뽑을 수 없는거였다.

[해결방안]
여기서 해결방안은 2가지로 줄였다.
1. 사용자에게 스캔본 대신 원본을 넣도록 한다.
2. OCR(이미지에서 글자를 인식하는 기술)을 추가한다.

하지만 사내 PC에서 간단히 돌릴 프로그램에 OCR 까지 넣는 부분은 과하다고 생각하여(물론 진짜 프로젝트 였다면 모든 방법을 찾아 진행했겠지만 별도 플로그램 설치에 한국어 인식 데이터까지 필요하다는 말에 결단이 필요했다.) 1번 방법을 선택하게 되었다.

이미지에서 텍스트를 읽어들이는 기능은 나중을 위한 숙제로 남겨 놓은거지 하지 않겠다는 말은 아니라고 여기에 남기겠다.

## GUI를 만들자!
아무리 클라이언트와 가까워도 그냥 파일 받아서 명령어 치세요. 라고 할 수 는 없었다.
그래서 사용자가 쉽게 쓸수있게 화면단을 만들기로 했다.
여기서 추천 받은건 'streamlit'이었다.

단순한 이유로 만들기 쉽다라는 이유로 선택했다.(꼭 어렵고 화려하다 보다는 쉽고 심플하다가 강점인 경우도 있기 마련이다... 라고 나는 생각한다.)

```
"""
app.py — 문서 검증 도구 GUI (streamlit)

중복 체크와 조견표를 브라우저 화면에서 클릭으로 실행한다.
모두 로컬에서 동작한다 (임베딩=fastembed, 요약/판정=Ollama).
실행: streamlit run app.py
"""

import os
os.environ["HF_HUB_DISABLE_SYMLINKS"] = "1"

import re
import json
import io
from pathlib import Path

import streamlit as st
from openpyxl import Workbook
from openpyxl.styles import Font, PatternFill, Alignment

import ollama
from rag import read_pptx, read_pdf, read_xlsx, get_embed_model, cosine_similarity


OLLAMA_MODEL = "qwen2.5:7b"


# ──────────────────────────────────────────────────────────────
# 공통: 파일 읽기
# ──────────────────────────────────────────────────────────────
def read_any(path):
    suffix = path.suffix.lower()
    if suffix == ".pptx":
        return read_pptx(path), "슬라이드"
    elif suffix == ".pdf":
        return read_pdf(path), "페이지"
    elif suffix == ".xlsx":
        return read_xlsx(path), "시트"
    return None, None


def split_sentences(text, min_chars):
    parts = re.split(r"[\n]+|(?<=[.!?。])\s+", text)
    return [p.strip() for p in parts if len(p.strip()) >= min_chars]


# ──────────────────────────────────────────────────────────────
# 엑셀을 메모리에 만들어 다운로드용 바이트로 반환
# ──────────────────────────────────────────────────────────────
def make_excel_bytes(headers, rows, widths, title):
    wb = Workbook()
    ws = wb.active
    ws.title = title
    ws.append(headers)
    for cell in ws[1]:
        cell.font = Font(bold=True, color="FFFFFF", name="맑은 고딕")
        cell.fill = PatternFill("solid", start_color="4472C4")
        cell.alignment = Alignment(horizontal="center", vertical="center")
    for r in rows:
        ws.append(r)
    for col, w in enumerate(widths, start=1):
        ws.column_dimensions[ws.cell(row=1, column=col).column_letter].width = w
    for row in ws.iter_rows(min_row=2):
        for cell in row:
            cell.alignment = Alignment(vertical="top", wrap_text=True)
    buf = io.BytesIO()
    wb.save(buf)
    return buf.getvalue()


# ──────────────────────────────────────────────────────────────
# 화면 시작
# ──────────────────────────────────────────────────────────────
st.set_page_config(page_title="문서 검증 도구", page_icon="📋")
st.title("📋 문서 검증 도구")
st.caption("모든 처리는 이 컴퓨터 안에서만 이루어집니다. 자료가 외부로 전송되지 않습니다.")

tab1, tab2 = st.tabs(["🔁 중복 체크", "📑 조견표 만들기"])


# ── 탭 1: 중복 체크 ──────────────────────────────────────────
with tab1:
    st.subheader("한 문서 안의 중복 문장 찾기")
    st.write("작성 중인 문서를 골라, 내부에서 비슷한 문장끼리 짝지어 봅니다.")

    up = st.file_uploader("검사할 문서", type=["pptx", "pdf", "xlsx"], key="dup")
    threshold = st.slider("유사도 기준 (높을수록 깐깐)", 0.70, 0.98, 0.90, 0.01)
    min_chars = st.slider("이 글자 수 미만 문장 제외 (소제목 거르기)", 5, 40, 25)

    if st.button("중복 체크 시작", key="dup_btn", type="primary"):
        if up is None:
            st.warning("먼저 문서를 올려주세요.")
        else:
            # 업로드 파일을 임시로 저장해 경로로 읽는다
            tmp = Path(up.name)
            tmp.write_bytes(up.getbuffer())
            blocks, label = read_any(tmp)

            if not blocks:
                st.error("문서에서 글자를 읽지 못했습니다. (스캔 PDF이거나 이미지일 수 있어요)")
            else:
                sentences, locations = [], []
                for idx, block in enumerate(blocks, start=1):
                    for s in split_sentences(block, min_chars):
                        sentences.append(s)
                        locations.append(f"{label} {idx}")

                st.info(f"문장 {len(sentences)}개를 추출했습니다. 후보를 추리는 중...")
                model = get_embed_model()
                vectors = list(model.embed([f"passage: {s}" for s in sentences]))

                pairs = []
                n = len(sentences)
                for i in range(n):
                    for j in range(i + 1, n):
                        score = cosine_similarity(vectors[i], vectors[j])
                        if score >= threshold:
                            pairs.append((score, i, j))
                pairs.sort(reverse=True)

                st.success(f"비슷한 문장 쌍 {len(pairs)}개를 찾았습니다.")

                if pairs:
                    rows = [[round(float(s), 3), locations[i], sentences[i][:300],
                             locations[j], sentences[j][:300]] for s, i, j in pairs]
                    # 화면 미리보기
                    st.dataframe(
                        [{"유사도": r[0], "위치A": r[1], "문장A": r[2],
                          "위치B": r[3], "문장B": r[4]} for r in rows],
```

## 그런데 화면을 만들어 제공해도 띄울려면 명령어가 필요한거 아냐?
물론 그랬다. 이 부분을 해결하기 위해 생각한 방법으로 **배치파일(.bat)**을 만들기로 했다.

배치파일을 만드는 동시에 해당 파일을 실행할때 라이브러리 설치도 하게 하고 싶었다.

그러기 위해 'requirements.txt'를 정리할 필요가 있었다.

해당 파일 정리 이후에 배치파일 실행시 라이브러리를 체크하는 로직을 넣고 없으면 설치하게 했다.

## 그냥 지나가나 했다... 또 문제 발생.
그렇게 배치파일을 만들고 실행했으나 동작하지 않았다. 화면 실행 파일을 직접 호출할 경우 정상적으로 나오기 때문에 배치파일에 문제가 있다고 판단했다.

원인은 인코딩의 문제였다.

배치파일을 메모장으로 열어 **ANSI**로 저장하자 한글이 다 깨지고 있었다.

그래서 가장 강력한 방법을 사용했다. 

**한글 내용 전부 삭제** 

전부 영어로 바꿔서 저장 한 후 실행시키자 문제없이 동작하는걸 확인했다.

## 누락 체크 추가 및 모델 변경 작업.

```
"""
logic_missing.py — 내 문서에 빠진 내용 찾기 (계산만, 화면 없음)

내 문서(A)를 기존 자료 폴더(B·C·D)와 비교해,
기존 자료엔 있는데 내 문서엔 없는 내용을 찾는다.

원리: 기존 자료의 각 조각을 A 전체에서 검색해,
가장 비슷한 것과의 유사도가 threshold보다 낮으면 '누락'으로 본다.
(중복 체크와 방향이 반대: 낮을 때 잡는다)
"""

from pathlib import Path

from rag import get_embed_model, cosine_similarity
from logic_common import read_any, summarize


def find_missing(my_path, ref_folder, threshold=0.70,
                 use_summary=True, progress_cb=None):
    """
    반환: (rows, a_count, ref_count)
      rows = [[최고유사도, 출처, 요약, 원문], ...] (유사도 낮은 순 = 확실히 빠진 것부터)
      a_count/ref_count = 내 문서/기존 자료의 조각 수
    my_path와 같은 이름의 파일은 기존 자료에서 제외한다(자기 자신 비교 방지).
    """
    my_path = Path(my_path)
    ref_folder = Path(ref_folder)

    # 1) 내 문서(A) 읽기
    a_blocks, _ = read_any(my_path)
    if not a_blocks:
        return None, 0, 0   # 읽기 실패 신호

    # 2) 기존 자료(B·C·D) 읽기 — 출처와 함께
    ref_blocks, ref_sources = [], []
    for path in ref_folder.rglob("*"):
        if path.name == my_path.name:   # 자기 자신 제외
            continue
        blocks, label = read_any(path)
        if not blocks:
            continue
        for idx, block in enumerate(blocks, start=1):
            block = block.strip()
            if block:
                ref_blocks.append(block)
                ref_sources.append(f"{path.name} / {label} {idx}")

    if not ref_blocks:
        return [], len(a_blocks), 0

    # 3) 벡터화 (A=passage=찾아지는 대상, 기존자료=query=A에서 찾을 질문)
    model = get_embed_model()
    a_vectors = list(model.embed([f"passage: {b}" for b in a_blocks]))
    ref_vectors = list(model.embed([f"query: {b}" for b in ref_blocks]))

    # 4) 기존자료 각 조각이 A에 있는지 — 최고 유사도가 낮으면 누락
    missing = []
    for k, rv in enumerate(ref_vectors):
        best = max(cosine_similarity(rv, av) for av in a_vectors)
        if best < threshold:
            missing.append((best, ref_sources[k], ref_blocks[k]))
    missing.sort()   # 유사도 낮은 순

    # 5) 요약 달기
    rows = []
    total = len(missing)
    for n, (score, src, text) in enumerate(missing, start=1):
        if use_summary and len(text) >= 15:
            summary = summarize(text)
        else:
            summary = text[:20]
        rows.append([round(float(score), 3), src, summary, text[:300]])
        if progress_cb:
            progress_cb(n, total)

    return rows, len(a_blocks), len(ref_blocks)
```
