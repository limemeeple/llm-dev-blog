왜 이 블로그를 시작했나#

Java 백엔드 개발을 해오다 LLM 엔지니어링으로 방향을 틀기로 했다. 새로운 분야의 첫 발을 떼는 만큼, 결과만 남기기보다 과정에서의 고민과 삽질을 기록으로 남기고 싶었다.

나중에 돌아봤을 때 “이때 이런 걸 몰랐었구나” 하고 웃을 수 있는 기록. 그리고 같은 길을 걷는 누군가에게 작게라도 도움이 되는 기록.

목표#

  • RAG의 동작 원리를 코드로 직접 구현해보기
  • 문서 기반 Q&A 시스템을 끝까지 완성하기
  • 검색 품질을 숫자로 측정하고 개선하기
  • 기존 Java 강점 + 새로 배우는 Python을 현실적인 구조로 결합하기

첫 프로젝트: 문서 기반 Q&A 시스템#

거창한 챗봇 데모가 아니라, “엔지니어링"이 보이는 프로젝트를 만들기로 했다.

레이어 언어 / 기술 역할
API · 서비스 Java (Spring Boot, Spring AI) 요청 라우팅, LLM 호출 오케스트레이션
데이터 · 평가 Python 청킹 · 임베딩 파이프라인, 평가 스크립트
저장소 벡터 DB (예정) 임베딩 검색

핵심은 단순히 “작동하는 RAG"에서 멈추지 않고, 검색 품질 개선의 before/after와 정량 평가까지 가는 것이다.

다음 할 일#

  1. Python으로 가장 단순한 RAG를 노트북 한 파일로 구현 (원리 체득)
  2. 이걸 Spring 서비스로 감싸 API화
  3. 평가셋을 만들고 정확도 측정
